## Introduction
Data Analysis Report by using [Power BI](https://www.microsoft.com/en-us/power-platform/products/power-bi) on the [Mobile Phone Sales dataset](https://app.enterprisedna.co/app/challenges/51) provided by [Enterprise DNA](https://enterprisedna.co/), in collaboration with Onyx Data for Practicing, Showcasing Data Visualization and Analytics Skills.
The dataset contains sales data of Mobile Phone sales starting from January 1, 2024 to the end of December 31, 2024 where it contains information about the type of phones sold from various channels like online, retail stores and partner stores. Including phones supplied by various companies like Samsung, Apple, OnePlus and Xiaomi. We are looking to pull and find insights from this data to help our business stake-holders to make better data driven decisions. 
The main feature of the this Report is the 4-4-5 Retail Calender implimentation along with the Time Series Analysis with the 4-4-5 Calendar and Standard Gregorian Calendar by using custom calendar table generated by using [DAX](https://learn.microsoft.com/en-us/dax/dax-overview). Please refer to the Date table section for more information.
## Dataset & Tables
This dataset was retrieved from the [Enterprise DNA Challenges](https://enterprisedna.co/challenges) page for practising and learning Power BI, following are the tables which were imported and created for this Report.
### Mobile_Sales Table
**Metadata:**
1. Transaction_ID: Unique ID for each recorded transaction
2. Transaction_Date: Date when the transaction was recorded
3. Mobile_Model: The model name of the Moble Device Sold
4. Brand: The name of the manufacturer of the Mobile Device
5. Storage_Size: Storage Size of the Mobile Device
6. Color: Color of the Mobile Device
7. Operating_System: Operating system that the Mobile Device uses
8. Price: Price of the Mobile Device
9. Units_Sold: Number of Mobile Device Units sold
10. Total_Revenue: Unit_Sold times Price
11. Sales_Channel: The outlet from where the mobile device was sold(Retial Store, Online or Partner Store)
12. Payment_Type: Method of payment utilized by the customers
13. Customer_Age: The age of the customer
14. Customer_Age_Group: The age group if the customer(6 Buckets/Bins of ages)
15. Customer_Gender: The gender of the customer
16. Country: The country in which the mobile device was sold
17. City: The city in which the mobile device was sold
18. Latitude: Latitude of the sales outlet
19. Longitude: Longitude of the sales outlet
20. Mobile SKU: Obtained by using DAX:
```
Mobile SKU =
Mobile_Sales[Brand] & "-" & Mobile_Sales[Mobile_Model] & "-" & Mobile_Sales[Color] & "-" & Mobile_Sales[Storage_Size] & "-" & Mobile_Sales[Operating_System]
```
### Date_Dim
This calendar is a custom calendar made using the standard 4-4-5 Calendar that Retail Business use to better track sales trends week by week which has 364 days, 52 weeks and 3 Quarters containing 2 months with 4 weeks and 1 month with 5 weeks arranged like 4-4-5 for each quarter. Lets look at the Metadata first and then we can look at the DAX.

**Metadata**
1. Date: Dates ranging from January 1, 2024 to December 31, 2024
2. Year: Year Index
3. Day of Year: Day number of the Fiscal Year 1 to 364
4. Quarter: Quarter number of the Fiscal Year
5. Week of Month: Week number of the Month(Two months containing 4 weeks and 1 month containing 5 weeks)
6. Month No in QTR: Month Number in Quarter, 1 to 3
7. Week: Week number of the Fiscal Year 1 to 52
8. Day of Week: Day number of the Week 1 to 7
9. Day of the Week Name: Day name of the week
10. G Day Number: Day Number of the year according to Standard Gregorian Calendar
11. G Month Number: Month Number of the year according to the Standard Gregorian Calendar
12. Fiscal Year: Fiscal Year Number 2024 and 2025
13. Fiscal Quarter: Fiscal Quarter Name like Q1, Q2, etc
14. Fiscal Year Quarter: 2024 FQ1, 2024 FQ2, ...
15. Fiscal Week: Fiscal Week name like FW1, FW2, etc
16. Fiscal Year Quarter Week: Like 2024 FQ1 FW1, 2024 FQ2 FQ2, etc
17. G Month Name: Name of the month according to the gregorian calendar
18. Fiscal Year Quarter Week Number: Number concatenated from: Fiscal Year + Quarter Index + Week Index

**Following is the DAX used for making the core of this table**
```
Date_Dim = 
VAR cal = CALENDARAUTO()

RETURN
ADDCOLUMNS(
    cal,
    "Year", QUOTIENT(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 364) + 1,
    "Day of Year", MOD(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 364) + 1,
    "Quarter", MOD(QUOTIENT(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 91), 4) + 1,
    "Week of Month", 
        VAR var1 = MOD(MOD(QUOTIENT(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 7), 52), 13)
        VAR var2 = IF(var1 = 12, 5, MOD(var1, 4) + 1)
        RETURN var2,
    "Month No in QTR",
        VAR var1 = MOD(MOD(QUOTIENT(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 7), 52), 13)
        VAR var2 = IF(var1 = 12, 3, QUOTIENT(var1, 4) + 1)
        RETURN var2,
    "Week", MOD(QUOTIENT(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 7), 52) + 1,
    "Day of Week", MOD(DATEDIFF(MINX(cal, [Date]), [Date], DAY), 7) + 1
)
```
**For rest of the Columns the DAX is following:**
Column Number 9:
```
Day of the Week Name = FORMAT ('Date_Dim'[Date], "dddd")
```
Column Number 10:
```
G Day Number = DATEDIFF(
    MINX(Date_Dim, Date_Dim[Date]),
    Date_Dim[Date],
    DAY
) + 1
```
Column Number 11:
```
G Month Number = MONTH(Date_Dim[Date])
```
Column Number 12:
```
Fiscal Year = Date_Dim[Year] + 2024 - 1
```
Column Number 13:
```
Fiscal Quarter = "FQ" & Date_Dim[Quarter]
```
Column Number 14:
```
Fiscal Year Quarter = Date_Dim[Fiscal Year] & " " & Date_Dim[Fiscal Quarter]
```
Column Number 15:
```
Fiscal Week = "FW" & Date_Dim[Week]
```
Column Number 16:
```
Fiscal Year Quarter Week = Date_Dim[Fiscal Year] & " " & Date_Dim[Fiscal Quarter] & " " & Date_Dim[Fiscal Week]
```
Column Number 17:
```
G Month Name = FORMAT(Date_Dim[Date], "MMMM")
```
Column Number 18:
```
Fiscal Year Quarter Week Number = Date_Dim[Fiscal Year] & "0" & Date_Dim[Quarter] & Date_Dim[Week]
```

## Measures
Here are the measures used for this Report:
### Base Measures
**Total Revenue**
```
Total Revenue = SUM(Mobile_Sales[Total_Revenue])
```
**Total Units Sold**
```
Total Units Sold = SUM(Mobile_Sales[Units_Sold])
```
### Time Intelligence Measure
Our calendar contains columns both for Retail 4-4-5 Calendar and Gregorian Calendar, and the Time Intelligence page contains Time intelligence based on both of the Calendars. Also we are not going to use the base time intelligence functions like TOTALYTD(), TOTALMTD() and TOTALQTD(). Since, we have a custom calendar we will need to make our custom measures for Time Series Analysis.
**Gregorian Calendar Month to Date Revenue**
```
Revenue G Month to Date = 
VAR CurrentGDay = MAX(Date_Dim[G Day Number])
VAR CurrentGMonth = MAX(Date_Dim[G Month Number])


RETURN
CALCULATE(
    [Total Revenue],
    FILTER(
        ALL(Date_Dim),
        Date_Dim[G Month Number] = CurrentGMonth &&
        Date_Dim[G Day Number] <= CurrentGDay
    )
)
```
**Units Sold G Month to Date**
```
Units Sold G Month to Date = 
VAR CurrentGDay = MAX(Date_Dim[G Day Number])
VAR CurrentGMonth = MAX(Date_Dim[G Month Number])


RETURN
CALCULATE(
    [Total Units Sold],
    FILTER(
        ALL(Date_Dim),
        Date_Dim[G Month Number] = CurrentGMonth &&
        Date_Dim[G Day Number] <= CurrentGDay
    )
)
```
**Revenue Week to Date ISO**
```
Revenue Week to Date ISO = 
VAR CurrentFiscalDay = MAX(Date_Dim[Day of Year])
VAR CurrentFiscalWeek = MAX(Date_Dim[Week])
VAR CurrentFiscalYear = MAX(Date_Dim[Year])

RETURN
CALCULATE(
    [Total Revenue],
    FILTER(
        ALL(Date_Dim),
        Date_Dim[Week] = CurrentFiscalWeek &&
        Date_Dim[Year] = CurrentFiscalYear &&
        Date_Dim[Day of Year] <= CurrentFiscalDay
    )
)
```
**Units Sold G Month to Date**
```
Units Sold G Month to Date = 
VAR CurrentGDay = MAX(Date_Dim[G Day Number])
VAR CurrentGMonth = MAX(Date_Dim[G Month Number])


RETURN
CALCULATE(
    [Total Units Sold],
    FILTER(
        ALL(Date_Dim),
        Date_Dim[G Month Number] = CurrentGMonth &&
        Date_Dim[G Day Number] <= CurrentGDay
    )
)
```
